# i18n

Minimalist i18n (≈ 1.5KB gzip). It does not replace `i18next` in massive apps;
it serves when you want 2-3 languages without extra weight.

## Setup

```tsx
import { I18nProvider } from "tempest-react-sdk";

const messages = {
  "pt-BR": {
    greet: "Olá, {name}",
    alos_one: "{count} Alô",
    alos_other: "{count} Alôs",
  },
  en: {
    greet: "Hi, {name}",
    alos_one: "{count} Alo",
    alos_other: "{count} Alos",
  },
};

<I18nProvider locale="pt-BR" fallbackLocale="en" messages={messages}>
  {children}
</I18nProvider>;
```

`I18nProvider` persists the choice in `localStorage["tempest-locale"]` and sets
`<html lang="…">` automatically.

## Usage

```tsx
import { useI18n } from "tempest-react-sdk";

const { t, plural, formatNumber, formatDate, setLocale, locale } = useI18n();

<p>{t("greet", { name: "Mau" })}</p>           // "Olá, Mau"
<p>{plural("alos", 3)}</p>                      // "3 Alôs"
<p>{formatNumber(1234.5, { style: "currency", currency: "BRL" })}</p>
<button onClick={() => setLocale("en")}>EN</button>;
```

Pluralization: suffix `_one` (count === 1) or `_other`. For rich plurals
(Russian, Polish), switch to `i18next` — it's not worth re-implementing
`Intl.PluralRules` here.

## Without React (imperative)

```ts
import { createI18n } from "tempest-react-sdk";

const i18n = createI18n({ locale: "pt-BR", messages });
i18n.t("greet", { name: "Mau" });
const en = i18n.withLocale("en");
```

## Dynamic loading

Load one JSON per locale and pass it to the provider. To split by route, swap
`messages` with a state and re-mount the provider — missing keys fall back.

```ts
const messagesPt = await fetch("/i18n/pt-BR.json").then((r) => r.json());
```
