# i18n

i18n minimalista (≈ 1.5KB gzip). Não substitui `i18next` em apps massivos; serve quando você quer 2-3 idiomas sem peso extra.

## Setup

```tsx
import { I18nProvider } from "tempest-react-sdk";

const messages = {
  "pt-BR": {
    greet: "Olá, {name}",
    alos_one: "{count} Alô",
    alos_other: "{count} Alôs",
  },
  en: {
    greet: "Hi, {name}",
    alos_one: "{count} Alo",
    alos_other: "{count} Alos",
  },
};

<I18nProvider locale="pt-BR" fallbackLocale="en" messages={messages}>
  {children}
</I18nProvider>;
```

`I18nProvider` persiste a escolha em `localStorage["tempest-locale"]`, seta `<html lang="…">` automaticamente.

## Uso

```tsx
import { useI18n } from "tempest-react-sdk";

const { t, plural, formatNumber, formatDate, setLocale, locale } = useI18n();

<p>{t("greet", { name: "Mau" })}</p>           // "Olá, Mau"
<p>{plural("alos", 3)}</p>                      // "3 Alôs"
<p>{formatNumber(1234.5, { style: "currency", currency: "BRL" })}</p>
<button onClick={() => setLocale("en")}>EN</button>;
```

Pluralização: suffix `_one` (count === 1) ou `_other`. Pra plurais ricos (russo, polonês), troque por `i18next` — não vale a pena re-implementar `Intl.PluralRules` aqui.

## Sem React (imperativo)

```ts
import { createI18n } from "tempest-react-sdk";

const i18n = createI18n({ locale: "pt-BR", messages });
i18n.t("greet", { name: "Mau" });
const en = i18n.withLocale("en");
```

## Carregamento dinâmico

Carregue um JSON por locale e passe pro provider. Pra split por rota, troque `messages` com um state e re-monte o provider — keys não encontradas caem no fallback.

```ts
const messagesPt = await fetch("/i18n/pt-BR.json").then((r) => r.json());
```
